# Excel 离线工具 - 全面测试方案

## 测试设计理念

从用户实际使用角度出发，覆盖以下维度：
1. **数据结构维度**: 单行表头、多行表头、合并单元格
2. **操作场景维度**: 拆分、合并、不同模式切换
3. **边界条件维度**: 空表头、大量表头行、表头行数变化
4. **格式保留维度**: 合并单元格、列宽、行高

---

## 测试场景矩阵

### 场景 1: 合并单元格表头
```
用户场景: 财务报表、统计表，表头有跨列合并
测试文件: merged-header.xlsx
结构:
  A      B      C      D      E
1 [======== 销售额 ========]  [== 利润 ==]
2 产品    Q1     Q2     Q3    Q1    Q2
3 产品A   100    200    300   10    20
4 产品B   150    250    350   15    25

关键测试点:
- !merges 属性是否正确复制
- 拆分后合并单元格是否保留
- 合并区域在列选择后是否正确调整
```

### 场景 2: 多行表头 (2-3行)
```
用户场景: 复杂报表，有多级分类
测试文件: multi-header-2rows.xlsx, multi-header-3rows.xlsx

2行表头:
  A        B        C        D
1 部门              第一季度
2 姓名     类别     1月      2月
3 张三     销售部   100      200

3行表头:
  A        B        C        D        E
1                    2024年
2         基本信息   第一季度   第二季度
3 姓名     部门     1月       4月
4 张三     销售部   100       150

关键测试点:
- 表头行数设置正确识别
- 数据行正确计算 (从第N+1行开始)
- 拆分后多行表头完整保留
```

### 场景 3: 表头行数变化
```
用户场景: 用户上传文件后修改表头行数设置
测试流程:
1. 上传多行表头文件
2. 默认表头行数=1，观察列识别
3. 修改表头行数为2，观察列识别变化
4. 修改表头行数为0（无表头），观察处理
5. 生成文件，验证表头处理正确

关键测试点:
- 表头行数变化时列列表实时更新
- 数据起始行正确计算
- 无表头模式正确处理
```

### 场景 4: 竖向拆分的合并单元格
```
用户场景: 竖向拆分保留固定列，但固定列包含合并单元格
测试文件: vertical-split-with-merge.xlsx

结构:
  A          B         C         D         E
1 [== 基本信息 ==]            销售额    利润
2 部门        姓名       类别     金额      金额
3 销售部      张三       A类      100       20
4 技术部      李四       B类      200       40

操作: 竖向拆分，选择固定列 A,B，数据列 C,D,E
关键测试点:
- 合并单元格在列重组后正确处理
- 部分合并区域被拆分时是否正常
```

### 场景 5: 合并文件的多表头一致性
```
用户场景: 合并多个文件，表头结构相同
测试文件: merge-header-1.xlsx, merge-header-2.xlsx

关键测试点:
- 多行表头文件合并时只保留一组表头
- 合并后表头完整性
- 不同表头行数的文件合并时的处理
```

## 测试用例详细设计

### TC-MERGE-01: 合并单元格保留测试
```
前置条件: 上传 merged-header.xlsx
操作步骤:
  1. 选择"按工作表拆分"
  2. 上传文件
  3. 选择工作表
  4. 生成文件
  5. 下载并验证

预期结果:
  - 生成的文件中合并单元格保留
  - !merges 属性正确
```

### TC-HEADER-01: 多行表头识别测试
```
前置条件: 上传 multi-header-2rows.xlsx
操作步骤:
  1. 选择"按列拆分(横向)"
  2. 上传文件
  3. 观察列列表显示
  4. 修改表头行数为 2
  5. 观察列列表更新

预期结果:
  - 默认显示第2行作为列名
  - 修改表头行数后列名正确更新
  - 数据行数计算正确
```

### TC-VERTICAL-01: 竖向拆分合并单元格测试
```
前置条件: 上传 vertical-split-with-merge.xlsx
操作步骤:
  1. 选择"按列拆分(竖向)"
  2. 上传文件
  3. 选择固定列(包含合并单元格)
  4. 选择数据列
  5. 生成文件

预期结果:
  - 固定列的合并单元格保留
  - 数据列正常拆分
  - 每个文件结构正确
```

## 验证点检查清单

### 格式保留
- [ ] 合并单元格 (!merges) 正确复制
- [ ] 列宽 (!cols) 保留
- [ ] 行高 (!rows) 保留
- [ ] 多行表头完整保留

### 数据处理
- [ ] 表头行数正确识别
- [ ] 数据起始行计算正确
- [ ] 无表头模式 (headerRows=0) 正常
- [ ] 列名在多行表头下正确拼接

### 边界条件
- [ ] 表头行数 > 实际行数时的处理
- [ ] 合并单元格部分被筛选时的处理
- [ ] 空表头单元格的处理
